# i18n Notes (v1)

## Supported Locales

- `zh-CN` (default)
- `en-US`

## Default Policy

- Web client defaults to `zh-CN` unless user has an explicit saved preference.
- Locale preference is persisted in browser local storage key `mwg_locale`.
- If user profile locale is introduced later, it can override bootstrap locale before first render.

## Scope of Translation

v1 translates these user-facing flows:

- Auth: guest/login/register/upgrade flow copy
- Lobby: room actions, matchmaking controls, moderation labels, history labels
- Room: participant controls, invite/report actions, game status labels, replay-adjacent controls
- Errors: network/API/WS error mapping to localized text
- Replay: playback controls and metadata

## Error Translation Strategy

- Known backend error codes/messages are mapped to localized UI copy.
- Unknown errors fall back to generic localized text and include original message in Chinese mode for operator visibility.

## Non-goals (v1)

- No server-side locale negotiation headers.
- No persisted locale profile field in DB yet.
- No per-message translation payload over realtime protocol.
